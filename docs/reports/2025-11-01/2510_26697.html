1. AutoDeco 核心架构与机制

AutoDeco 的核心思想是在标准的变换器（Transformer）模型之上，增加轻量级的预测头（Prediction Heads）——一个温度头（temperature head）和一个 top-p 头。这些预测头（通常是简单的2层MLP）在每个解码步骤中，利用模型当前的隐藏状态来动态预测下一个 token 所需的最佳采样参数（即温度和 top-p 值）。

  • 工作流程：在生成过程中，标准的语言模型头计算 logits（逻辑值），而 AutoDeco 的预测头则并行地预测动态参数。具体来说：
    1. 温度预测：温度参数直接从隐藏状态中预测。
    2. Top-p 预测：top-p 参数则利用隐藏状态和刚刚预测出的温度作为输入进行计算。这种微观上的依赖性使得两个参数之间能够进行更细腻的交互。
  • 内部概率修改：模型随后使用这些动态预测出的参数，对内部的 logits 分布进行实时重标定和筛选，生成最终的、经过动态调整的概率分布，并从中采样下一个 token。

2. 训练策略：可微分的端到端优化

训练 AutoDeco 的主要挑战在于，采样参数（如温度和 top-p）缺乏逐字对应的“真实标签”，并且标准的 top-p 采样算法因其“硬截止”特性而不可微分，这会中断梯度的反向传播。

为了解决这个问题，论文提出了一种新颖的、可微分的“软”top-p 机制，从而实现了完全的端到端优化：

  1. 温度缩放概率：首先，使用预测的 logits $l$ 和温度 $\hat{T}$ 计算初始概率分布 $p$： $p = \text{softmax}\left(\frac{l}{\hat{T}}\right)$

  2. 生成可微分掩码：在对概率排序并计算累计概率 $c$ 后，生成一个“软掩码” $m(\text{sorted})$，它平滑地衰减超出 top-p 阈值 $\hat{P}$ 的 token 的概率： $m(\text{sorted}) = \exp\left(-\alpha \cdot \text{ReLU}(c - \hat{P})\right)$ 其中，$\alpha$ 是控制衰减陡度的超参数。

  3. 最终概率分布：将软掩码应用到初始概率上并重新归一化，得到最终的可微分分布 $p^{\sim}$： $p^{\sim} = \frac{p \odot m}{\sum(p \odot m) + \epsilon}$

通过这个可微分的管道，整个训练过程简化为在最终分布 $p^{\sim}$ 和真实 token $y^*$ 之间计算标准的交叉熵损失。梯度可以无障碍地反向传播，从而同时更新基础模型和 AutoDeco 预测头的参数。

此外，为了增强模型的鲁棒性，还采用了去偏训练策略： * 简单令牌掩码：对于模型可以轻易预测正确的“简单”令牌，随机掩码其训练损失（例如60%），迫使模型从更具挑战性的样本中学习。 * 动态微调：重新加权训练损失，使模型更关注其具有合理先验知识的令牌，从而学会在经过校准的不确定性情况下更明智地应用高温。

3. 推理过程：高效的动态解码

AutoDeco 的设计在推理时极为高效。所有动态调整都无缝集成到模型的标准前向传播中，几乎不增加额外的计算负担。 * 效率：由于预测头非常轻量，其引入的计算开销（FLOPs 和内存）微乎其微，总生成延迟仅增加 1-2%。 * 易用性：这使得 AutoDeco 可以作为一个即插即用（plug-and-play）的直接替换方案，用户无需修改现有的生成逻辑。
