完整解决方案：LIVE-SWE-AGENT——一个能够自我进化的实时软件工程代理

引言：核心理念

论文提出的核心解决方案是 LIVE-SWE-AGENT，一个创新的软件工程智能代理。其核心思想是赋予代理在解决软件问题的过程中实时自我进化（On-the-fly Self-Evolution）的能力。与传统依赖固定工具集或需要昂贵离线训练的代理不同，LIVE-SWE-AGENT 能够在运行时根据遇到的具体问题，动态地创建、修改和扩展自身的能力，尤其是其工具集。

关键机制与实现流程

LIVE-SWE-AGENT 的自我进化能力依赖于以下几个紧密相连的关键机制：

1. 最小化基础架构与在线进化
  • 起始状态: 代理从一个极其简单的基础架构开始，通常只提供对基础 bash 命令的访问权限（例如，基于 mini-SWE-agent 框架）。它没有预先设计的复杂工具集。
  • 在线进化而非离线训练: 该代理的设计完全摒弃了昂贵的离线训练过程。它的所有能力进化都发生在解决实际问题的“在线”过程中。新能力的形成源于代理从遇到的具体问题中学习，而非在固定的基准测试上进行预训练，这使其具有更强的适应性和通用性。
2. 动态工具创建与合成 (Dynamic Tool Synthesis)

这是 LIVE-SWE-AGENT 实现自我进化的核心手段。代理能够根据任务需求，自主合成自定义工具。

  • 工具的定义: 自定义工具被实现为可以在环境中执行的脚本（例如 Python 脚本）。这种设计非常直观：创建工具即是编写一个脚本文件，使用工具即是执行该脚本。
  • 工具的类型:
    • 通用工具: 代理可以创建提高通用工作流程效率的工具。例如，一个定制的代码搜索工具，可以自动忽略无关目录（如 __pycache__, .git）并显示匹配代码的上下文；或是一个复杂的 Go 文件分析器（go_analyzer.py），用于查找结构体、函数定义和引用。
    • 问题特定工具: 代理还能创建仅用于解决当前特定问题的专用工具。例如，在处理一个涉及 MARC 文件（一种二进制格式）的问题时，代理创建了一个专门的分析工具，将二进制内容以人类可读的格式打印出来，帮助其理解问题并评估解决方案。
  • 优势: 相比使用复杂的 bash 命令链，这些自定义工具目的明确、易于使用，并且能够提供更清晰的反馈（例如，报告替换操作是否成功），避免了因缺乏反馈而导致的误导。
3. 反思机制与迭代决策 (Reflection Mechanism)

为了让工具创建更加智能和有目的性，LIVE-SWE-AGENT 引入了一个轻量级的反思机制。

  • 触发方式: 在代理的每个操作循环中，当环境返回结果后，系统会附加一个简单的反思提示（Step Reflection Prompt），明确地询问代理：“创建一个新工具或修改一个现有工具是否能加速解决当前问题？”
  • 作用: 这一机制将“创建工具”提升为一个与“运行测试”或“编辑文件”并列的显式决策点。它促使代理在每一步都评估自身能力，并思考如何通过创造新工具来克服障碍，从而显著提高了有用工具的创建频率和针对性。
4. 交互式工作流程

代理的每一步操作都遵循一个结构化的交互模式，以确保思维的清晰性和操作的准确性。

  • “THOUGHT” + “bash command” 结构: 在执行任何操作前，代理必须首先输出一段 THOUGHT 文本，阐述其思考过程和下一步的目标。随后，它会提供一个可执行的 bash 命令来完成该目标。
  • 推荐工作流程: 系统建议代理遵循一个标准的问题解决流程：分析代码库 -> 创建脚本复现问题 -> 编辑源代码 -> 运行脚本验证修复 -> 测试边界情况。

性能评估

LIVE-SWE-AGENT 在多个行业标准基准测试中展示了其卓越的性能：
