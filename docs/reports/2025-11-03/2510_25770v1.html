本文旨在解决生成模型（特别是大语言模型LLMs）输出正确性的评估问题。这是一个重要且持续存在的问题，因为LLMs在广泛应用中容易产生错误输出（即“幻觉”），而现有的评估机制（如基于p值/p分数的方法）在提供可靠的统计保证方面存在局限，尤其是在处理用户可变的容忍度（α）和一种被称为“大小失真”（size distortion）的后验错误概念时。该问题在数学推理和属性遵循（如真实性、指令遵循）等关键应用场景中尤为突出。

解决方案

这篇论文提出了一种名为e-scores的新型评估框架，旨在为生成模型（特别是大语言模型LLM）的输出提供可靠的统计保证，以判断其正确性。该方法的核心优势在于克服了传统p值方法的局限性，允许用户在观察到模型输出后，灵活地调整错误容忍度（α），从而有效避免p-hacking等问题。

以下是该解决方案的详细构成：

1. 核心概念：e-scores的定义与原理

e-scores是一种基于统计学中e-values（期望值）理论的评分机制，用于量化生成模型输出的错误程度。其基本原理如下：

  • 评分解释：每个生成的响应（y）都会被赋予一个e-score。分数的高低直接反映了响应的正确性：
    • 低e-score：表示该响应是正确的。
    • 高e-score：表示该响应是错误的。
  • 统计保证：该方法提供了一个关键的统计保证：在用户设定的任意容忍度α下，最终筛选出的响应集中包含错误响应的概率不会超过α。其目标是满足以下不等式： \[ P \{ \exists (Y, \cdot) \in Sα (X, gπ (X)) \text{ s.t. } o (X, Y) = 0 \} \leq α \] 其中，Sα是经过e-score筛选后得分小于等于α的响应集，o(X, Y) = 0 表示响应Y是错误的。

2. 关键创新：后验容忍度调整的灵活性

传统统计方法（如p值）要求在分析数据前预设一个固定的容忍度α，这在实践中缺乏灵活性。e-scores通过引入“大小失真”（size distortion）这一后验错误概念，实现了数据依赖的后验容忍度（post-hoc α）调整。

  • 工作流程：用户无需预先承诺一个α值。他们可以先生成响应并计算出所有响应的e-scores，然后根据这些分数的分布情况，再决定一个合适的α值来过滤响应。
  • 两种后验容忍度策略：
    1. 最大约束自适应α：用户可以将容忍度α动态更新为响应集中的最大e-score，从而确保所有生成的响应都被考虑在内，同时了解当前输出质量下的最大风险。
    2. 分数包含策略：用户可以选择包含特定比例（如前80%）的响应，此时容忍度α则设定为该比例下响应的最高e-score。

这种后验调整机制赋予了用户极大的灵活性，可以根据具体任务的需求和风险偏好，动态地管理错误率。

3. 实现细节：e-scores的计算方法

e-scores的计算依赖于一个独立的“神谕”估计器（oracle estimator） ô，它是一个预训练模型，用于预测一个响应正确的概率（输出在之间）。

  • 计算公式：e-score的具体计算公式借鉴了符合预测（conformal prediction）的思想，其核心形式如下： \[ se\text{-}score(x{n+1}, y{n+1}) = (n + 1) \cdot \frac{f(x{n+1}, y{n+1})}{f(x{n+1}, y{n+1}) + \sum{i=1}^{n} f^{*}(x{i}, O(x{i}, g{\pi}(x_{i})))}^{-1} \] 其中，n 是校准数据集的大小，f 是一个将神谕估计器ô的预测值进行转换的函数。
  • 转换函数选项：论文提出了多种转换函数 f，以适应不同的评估需求，例如：
    1. \$f^{o}(x, y) = \hat{o}(x, y)\$
    2. \$f^{o}(x, y) = (1 - \hat{o}(x, y))^{-1}\$
    3. \$f^{o}(x, y) = \hat{o}(x, y) \cdot (1 - \hat{o}(x, y))^{-1}\$
  • 综合e-score：当有多个基础e-score时（例如针对多选项任务），可以通过简单的平均化处理将它们合并成一个综合e-score，以提供更稳健的评估结果。

4. 实验验证与应用场景

论文通过实验展示了e-scores在评估大语言模型输出时的有效性，主要集中在以下两个场景：

  1. 数学事实性：评估LLM在解决数学推理问题时，其输出步骤和答案的正确性。
  2. 属性约束满足：使用UltraFeedback数据集，检查生成的文本是否满足用户的特定期望属性（如真实性、诚实性、指令遵循性、有用性）。
  • 实验模型：实验中使用了Qwen2.5-Math-7B-PRM800K (QwenPRM) 和 Math-Shepherd-PRM-7B (MathShepherdPRM) 等预训练LLM作为“神谕”估计器。
  • 评估指标：实验的核心评估指标是大小失真（期望其均值≤1），同时还考察了错误率与α的关系、精确率和召回率。
  • 实验结论：实验结果表明，e-scores能够有效地将大小失真控制在1以下，保证了统计有效性。相比之下，传统的p-score方法在某些情况下无法做到这一点。

5. 解决方案的优势总结

  • 避免p-hacking：通过后验容忍度调整，从根本上解决了传统p值方法因预设α而导致的脆弱性。
  • 广泛的适用性：理论上保证了该方法适用于任何生成模型和更广泛的响应集类型。
  • 计算高效：e-score的计算是一次性的总和运算，其时间和内存复杂度为常量（O(1)），远优于p-score随校准数据量线性增长的复杂度（O(n)）。
  • 提升用户体验和控制力：用户可以根据模型输出的实际质量动态调整评估标准，从而获得更符合实际需求的反馈和更可靠的结果。
